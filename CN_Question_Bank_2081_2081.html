Question 2

What are the features of the application layer? Why is DNS required? Explain about recursive, non-recursive, and iterative DNS queries.

Answer:

The features of the application layer are:

  • End-to-End Communication: Makes it possible for software programs running on various devices to interact.
  • Data presentation: Data presentation makes ensuring that the data is structured correctly and that the apps that receive it can understand it.
  • Protocol Support: Offers many applications HTTP, FTP, SMTP, and DNS services.
  • Resource Sharing: Files, printers, and other resources may be shared by networked devices thanks to resource sharing.
  • User Interface: Enables users to engage with programs on the network.

The Domain Name System (DNS) is an essential part of the internet that converts domain names into IP addresses so that users may access websites and other resources:

  • Eliminates the need to memorize IP addresses:
    It would surely be awful to have to memorize the IP addresses of all the websites one frequently accesses, including Google, Facebook, Twitter, and so on. DNS servers are a creative approach to convert domain or subdomain names to IP addresses.
  • Increasing the speed of internet connections:
    DNS speeds up internet by caching, load balancing, using nearby servers, efficient hierarchy, and pre-fetching queries.
  • Scalability:
    DNS effectively arranges and maintains millions of domain names, facilitating the expansion of the internet.
  • Load balancing enabled:
    To balance the traffic and prevent any one server from becoming overburdened, the DNS server might prioritize or cycle these addresses.

Recursive DNS Query: A DNS server receives a query from the DNS resolver (client) and is responsible for contacting further DNS servers until it receives a response or returns an error. The customer awaits the outcome.

Non-Recursive DNS Query: When a DNS resolver sends a query to a DNS server, the server does not query other servers; instead, it just returns the best response it has (either authoritative or cached). It returns an error if it is unsure.

Iterative DNS Query: The DNS resolver sends a query to a DNS server, and if the server is unable to respond, it provides the resolver with the address of another DNS server to try. Until it receives an error or the answer, the resolver keeps asking questions.

Question 3

Provide reasons for transitions from IPv4 to IPv6. Describe the IPv6 datagram format.

Answer:

Reasons for Transition from IPv4 to IPv6:

  1. Resolve the Space Limitation:
    The 32-bit addresses used by IPv4 provide over 4.3 billion distinct addresses, however this is inadequate given how quickly the internet and linked devices are expanding. An address space that is much bigger is made possible by IPv6’s 128-bit addresses.
  2. Enhanced Security:
    IPv6 improves data security by virtue of its integrated support for IPsec (Internet Protocol Security) for authentication and encryption.
  3. Optimal Routing:
    The size of routing tables is decreased by IPv6’s hierarchical addressing scheme, which streamlines routing.
  4. Removal of NAT:
    IPv6’s large address space eliminates the requirement for Network Address Translation (NAT), making end-to-end communication possible and streamlining network administration.

 

fig: IPv6 datagram format

The IPv6 datagram has a fixed header of 40 bytes and an optional extension header. The structure is as follows:

  1. Version (4 bits):
    • The IP version is specified (6 for IPv6).
  2. Traffic Class (8 bits):
    • It is used to distinguish between traffic priority for QoS.
  3. Flow Label (20 bits):
    • It finds data flows that need to be handled differently, such real-time streaming.
  4. Payload Length (16 bits):
    • The payload’s size, including data and extension headers, is shown in bytes.
  5. Next Header (8 bits):
    • Specifies the type of the next header (e.g., TCP, UDP, or an extension header).
  6. Hop Limit (8 bits):
    • It replaces the IPv4 Time-to-Live (TTL) field, which indicates the most hops a packet is allowed to make.
  7. Source Address (128 bits):
    • The IPv6 address of the sender.
  8. Destination Address (128 bits):
    • The IPv6 address of the recipient.

Question 5

Illustrate the functionality of Media Access Control.

Answer:

Media Access Control (MAC) is a sublayer of the Data Link Layer. Among its features are the following:

  • It is in charge of encapsulating frames so they may be sent across the LLC and higher levels of the OSI network, providing an abstraction of the physical layer.
  • It fixes the source station’s and destination station’s (or groups of destination stations’) addressing.
  • When more than one data frame has to be sent, it carries out multiple access resolutions. It chooses the transmission channel access techniques.
  • It also creates frame check sequences, which helps prevent transmission mistakes; it also carries out collision resolution and starts retransmission in the event of collisions.
  • Verifies data integrity using techniques like Cyclic Redundancy Check (CRC).
  • Ensures devices are identified using unique MAC addresses for proper data delivery.

Question 6

Why do you think Network Functions Virtualization architecture is required? Explain.

Answer:

Network services like as load balancers, routers, and firewalls are isolated from proprietary hardware by Network services Virtualization (NFV). The software that performs these tasks runs on general-purpose servers in virtual machines or containers. This design increases the network’s ability to adapt to changing needs, optimizes resource usage, and lessens reliance on particular technology. Network Functions Virtualization architecture is required because of the following reason:

  1. Cost Reduction: By substituting virtualized services running on common servers for specialized, costly hardware, NFV lowers the requirement for such gear.
  2. Flexibility and Agility: It makes it simpler to scale up or down as needed by enabling the quick deployment of network services.
  3. Simplified Management: By allowing for central management of virtualized network operations, complexity may be decreased.
  4. Faster Innovation: Without requiring hardware modifications, NFV allows for rapid upgrades and the deployment of new services.
  5. Energy Efficiency: Less energy is used when network operations are consolidated onto fewer hardware platforms.

Assume a class C network and divide it into three subnets. What is the value of the new subnet?

Answer:

A Class C network has a default subnet mask of 255.255.255.0 (or /24) additionally, since 22 = 4 subnets, you need at least 2 bits to generate 3 subnets.

We need to divide the network into 3 subnets. To do this, we need to borrow bits from the host portion so, by borrowing 2 bits, the new subnet mask becomes 255.255.255.192 (or /26).

It is possible to build four subnets using a subnet mask of /26 (22 = 4). However, only three subnets are required and the other can be left unused.

Assuming the original network is 192.168.1.0/24, we may compute the subnets for a Class C network to view the subnets:

  1. 192.168.1.0/26 is the first subnet where broadcast: 192.168.1.62, useable range: 192.168.1.1 to 192.168.1.62
  2. 192.168.1.64/26 is the second subnet where broadcast: 192.168.1.127, useable range: 192.168.1.65 to 192.168.1.126
  3.  192.168.1.128/26 is the third subnet where broadcast: 192.168.1.191, useable range: 192.168.1.129 to 192.168.1.190
  4. 192.168.1.192/26 is the fourth subnet where broadcast: 192.168.1.255, useable range: 192.168.1.193 to 192.168.1.254

Hence, the new subnet mask is 255.255.255.192 (or /26), allowing for 4 subnets with 62 usable IP addresses per subnet.

Token Generation: The permitted data transmission rate is represented by the set rate at which tokens are added to the bucket.

Bucket Capacity: The bucket can’t hold more tokens than it can hold because of its maximum capacity.

Question 11

Explain any two wireless transmission media.

Answer:

Wireless transmission media describes communication technologies that send signals or data without the need of actual connections or wires. Instead, they use electromagnetic waves like microwaves, radio waves, or infrared to make it possible to send data across long distances. Wi-Fi, Bluetooth, and satellite communication are a few examples.

Any two wireless transmission media are:

a. Radiowaves:
Long-distance wireless communication is made possible by radio waves, a form of electromagnetic wave. Their ability to travel great distances and pass through barriers like walls and buildings makes them popular for wireless communication. They are appropriate for broadcasting and mobile communication because to their low frequency and ability to pass through barriers. Its applications include Wi-Fi networks, television transmission, and FM/AM radios.

Characteristics:

  • Frequency Range: From 3 Hz to 300 GHz.
  • Propagation: Can travel through air, vacuum, and even some solid materials.

b. Microwaves:
Microwaves are a specific type of electromagnetic wave with frequencies ranging from 1 GHz to 300 GHz, or wavelengths between 1 millimeter and 30 centimeters.Using high-frequency electromagnetic waves, microwaves provide point-to-point communication. They need a direct line of sight and are less susceptible to interference, among other features. Its applications include cellular networks, bluetooth gadgets, satellite communication, point-to-point communication lines, and other wireless communication technologies all make substantial use of them.

Characteristics:

  1. Line-of-Sight Propagation: Microwaves need unhindered, uncluttered routes to transmit since they normally move in straight lines.
  2. High Bandwidth: Because they can handle high data transfer rates, they may be used for broadband communication.
  3. Specific frequency bands, such as the L, S, C, X, and Ku bands, are used by microwaves for a variety of purposes.

 

Question 12

Write short notes on:

  1. ICMP
  2. IGMP
Answer:

a. ICMP (Internet Control Message Protocol) :
Internet Control Message Protocol (ICMP) is a network layer protocol that is used in IP networks for diagnostic and problem reporting purposes. It facilitates the communication of network conditions and error messages (such as “unreachable destination”) between devices. Echo Request/Reply, which is utilized by ping, and Destination Unreachable are examples of common ICMP messages. It helps with debugging and guarantees improved network traffic management.

Internet Control Message Protocol (ICMP) is used for error reporting. Error Reporting by ICMP works by sending messages to the sender from the receiver in the case when data is not received.

b. IGMP (Internet Group Management Protocol) :
In order to communicate with IP networks via multicasting, hosts and neighboring routers employ the IGMP protocol, which effectively leverages resources to send data packets and messages. Because multicast communication can involve one or more senders and receivers, IGMP can be utilized in online conferencing tools, gaming, and video streaming. This protocol may be used on IPv4 networks and IPv6 networks.

In order to control multicast group memberships on an IP network, IGMP is required. It guarantees that multicast traffic is only routed to hosts that are interested in receiving it and enables routers to identify which hosts are members of certain multicast groups. This lessens network congestion.
